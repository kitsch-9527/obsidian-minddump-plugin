# Jot 插件更新能力需求文档

## 1. 背景

当前 Jot 插件仅支持新增记录，不支持在 Jot 视图中直接编辑已有记录。用户希望：

- 直接修改已有记录，不再通过“新建一条”来模拟更新
- 直接给已有记录加标签或删标签
- 修改后自动记录更新时间，并支持后续查询

## 2. 现状与问题

### 2.1 当前保存逻辑是“新增”

- `saveJot()` 每次都会生成当前时间戳，并调用保存函数写入新条目
- `saveToMultiFile()` / `saveToSingleFile()` 仅做插入，不做替换
- 结果是“更新旧记录”在行为上会变成“新增新记录”

### 2.2 当前 UI 缺失编辑动作

- Jot 卡片点击行为是跳转到源文件
- 卡片缺少“点击即编辑”的主交互设计，编辑流程不直观
- 无“长按打开原文件”的替代入口设计，编辑与跳转行为易冲突

### 2.3 数据模型无更新时间字段

- 目前 `Jot` 结构中只有 `date` 与 `time`（创建时间语义）
- 无 `updatedAt` 字段，无法按“最后修改时间”查询

## 3. 目标

本期目标如下：

1. 支持在 Jot 视图中直接编辑记录内容（原地更新，不新增）
2. 支持在 Jot 视图中直接增删标签
3. 修改后写入并展示更新时间
4. 支持按更新时间过滤查询
5. 保持对历史记录格式兼容

## 4. 范围

### 4.1 In Scope

- 单条记录编辑（内容、标签、来源）
- 编辑后更新时间维护
- 查询语法扩展：`updated:YYYY-MM-DD`
- 多文件模式与单文件模式均支持

### 4.2 Out of Scope

- 批量编辑
- 协同冲突可视化处理
- 高级查询语法（如复杂区间表达式）作为后续迭代

## 5. 功能需求

## 5.1 FR-1 编辑入口与交互

- 点击记录卡片：直接进入编辑态
- 长按记录卡片：打开对应源文件
- 进入编辑态后可修改：
  - 内容（Markdown 文本）
  - 标签（新增、删除）
  - 来源（可选）
- 提供 `保存` 与 `取消` 操作

## 5.2 FR-2 原地更新

- 保存编辑时必须定位并替换原记录，不创建新记录
- 成功后即时刷新列表和过滤结果
- 失败时提示错误，并保留原始记录不破坏

## 5.3 FR-3 标签编辑

- 支持直接加标签、删标签
- 标签保存前标准化：
  - 去掉 `#` 前缀再存储
  - 去重
  - 去空值

## 5.4 FR-4 更新时间维护

- 记录创建时：`createdAt = now`，`updatedAt = now`
- 记录编辑时：`createdAt` 不变，`updatedAt = now`
- UI 至少展示 `updatedAt`

## 5.5 FR-5 更新时间查询

- 最小可用查询：`updated:YYYY-MM-DD`
- 查询行为：
  - 仅显示 `updatedAt` 日期匹配的记录
  - 可与标签过滤联动

## 6. 数据格式设计

## 6.1 是否可用 `####` 存储更新时间

结论：**可以使用 `####` 存储，但必须采用固定键名格式，避免被识别为普通正文。**

推荐格式（每条记录）：

```markdown
### 2026-04-30 10:15:00
#### id: 20260430-101500-abc123
#### updatedAt: 2026-04-30 11:02:45

这里是记录正文...

#tag1 #tag2

来源: example

---
```

## 6.2 使用 `####` 的理由与约束

- 当前解析记录边界依赖 `### ` 行；`#### ` 不会被当成新记录开始
- 但若不改解析器，`#### updatedAt:` 会被当普通内容行
- 因此必须新增元数据解析规则，显式识别：
  - `#### id: <value>`
  - `#### updatedAt: <YYYY-MM-DD HH:mm:ss>`

## 6.3 记录唯一标识

- 新增 `id`（强制）
- 定位与更新优先按 `id`，避免仅按时间戳命中错误记录

## 7. 兼容与迁移

## 7.1 历史数据兼容

- 历史记录无 `id`/`updatedAt` 时仍需可读
- 加载历史记录时默认：
  - `id` 可临时回填（内存计算）或首次编辑落盘补齐
  - `updatedAt = createdAt`

## 7.2 渐进迁移策略

1. 先支持读取新旧两种格式
2. 用户首次编辑旧记录时，自动写入 `id` 与 `updatedAt`
3. 不强制一次性全量迁移

## 8. 验收标准（MVP）

1. 在 Jot 视图点击记录可直接进入编辑态并保存到原记录
2. 在 Jot 视图长按记录可打开对应源文件
3. 可直接为已有记录增删标签并实时生效
4. 每次编辑后 `updatedAt` 正确变化
5. `updated:YYYY-MM-DD` 查询可用
6. 历史记录无元数据时不报错且可编辑

## 9. 风险与注意事项

- 若继续按 `### 日期时间` 定位，存在重名/误命中风险，需引入 `id`
- 解析器需明确区分正文中的普通 `####` 标题与系统元数据行
- 多文件与单文件模式都需覆盖更新逻辑和测试

## 10. 后续实现建议

- 优先实现数据层（解析 + 定位 + 原地替换）再接 UI 编辑态
- 为更新时间查询单独增加过滤分支，保持与现有 `date:` 查询一致风格
- 增加最小回归测试：新增、编辑、查询、历史兼容
