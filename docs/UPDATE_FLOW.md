# 随手记更新流程分析

## 概述

本文档分析 Jot 插件中"更新随手记"的数据流和行为。

## 核心发现：插件不支持原地更新

**重要**: Jot 插件**没有**原地更新/编辑功能。当你在今天"更新"昨天创建的随手记时，实际上是**创建了一条全新的随手记**，原记录保持不变。

### 验证依据

1. **saveJot 函数** (main.ts:151-194) — 每次调用都生成新的时间戳
   ```typescript
   const now = new Date();
   const dateStr = moment(now).format("YYYY-MM-DD");
   const fullDateTime = moment(now).format("YYYY-MM-DD HH:mm:ss");
   ```

2. **saveToMultiFile / saveToSingleFile** — 仅执行插入操作，不修改现有内容
   - 新记录插入位置: frontmatter 之后、原有内容之前
   - 使用 `vault.process()` 原子性写入
   - **没有任何编辑或删除逻辑**

3. **view.ts** — Jot 卡片点击事件仅跳转到源文件
   ```typescript
   card.addEventListener("click", () => this.openJot(jot));
   ```
   - 无编辑按钮
   - 无删除按钮

## 场景分析：昨天创建 → 今天更新

### 用户操作流程

1. **昨天** (2026-04-29) 创建随手记
   - 输入内容 → 保存
   - 系统生成时间戳: `### 2026-04-29 18:30:00`
   - 保存到文件: `Jots/jot-20260429.md`

2. **今天** (2026-04-30) 想要"更新"该随手记
   - 用户可能在源文件中手动编辑
   - **或者**在 JotView 中重新输入内容 → 保存 = 创建新记录！

### 数据文件变化

**昨天保存后** — `Jots/jot-20260429.md`:
```markdown
### 2026-04-29 18:30:00

昨天的随手记内容

#tag1

来源: 来源信息

---
```

**今天重新保存后** — `Jots/jot-20260430.md` (新建):
```markdown
### 2026-04-30 10:15:00

更新后的内容（实际上是新建！）

#tag1 #tag2

---
```

**原记录不变** — `Jots/jot-20260429.md`:
```markdown
### 2026-04-29 18:30:00

昨天的随手记内容

#tag1

来源: 来源信息

---
```

### UI 显示

在 JotView 中，今天会显示两条随手记:
- 新的 (2026-04-30 10:15:00) — 显示在列表顶部
- 旧的 (2026-04-29 18:30:00) — 按时间排序显示在下方

## 数据流图

```
用户输入内容
    ↓
[saveJot()]
    ↓
生成新时间戳 (当前时刻)
    ↓
logMode === "multi"
  ? [saveToMultiFile()] → jot-YYYYMMDD.md
  : [saveToSingleFile()] → jots.md
    ↓
vault.process() 插入新条目
    ↓
原记录 UNCHANGED (无编辑/删除)
```

## 如何真正"更新"一个 Jot

目前有两种方式:

### 方式 1: 在源文件中手动编辑

1. 点击 Jot 卡片 → `openJot(jot)` 打开源文件并定位
2. 在 Obsidian 编辑器中直接修改内容
3. **缺点**: JotView 列表不会自动刷新，需手动刷新

### 方式 2: 通过文件系统直接编辑

1. 使用其他编辑器修改 `Jots/` 目录下的 `.md` 文件
2. **缺点**: 同上，UI 不同步

## 缺失的功能

- [ ] 原地编辑随手记
- [ ] 删除随手记
- [ ] 编辑后 UI 自动刷新
- [ ] 更新时保留原创建时间的能力

## 建议

如果需要"更新"功能，可以考虑:

1. **扩展 Jot 数据模型** — 添加 `id` 字段支持唯一标识
2. **添加编辑模式** — 在 view.ts 中为 Jot 卡片添加编辑按钮
3. **原地更新算法** — 修改 `saveToMultiFile` 支持按 ID 查找并替换
4. **软删除标记** — 添加 `deleted` 字段而非物理删除
