# Jot 快速记录图片粘贴需求文档

## 1. 背景

当前 Jot 的快速记录（Capture / 快速输入区域）支持文本输入与附件上传，但不支持直接 `Ctrl/Cmd + V` 粘贴图片。用户在截图后需要额外手动保存或手动选择文件上传，操作链路长、打断记录效率。

本期希望补齐“直接粘贴图片”能力，并统一走现有附件上传机制，保证命名、路径、错误处理与附件渲染行为一致。

## 2. 目标

本期目标如下：

1. 在快速记录输入区支持直接粘贴图片
2. 粘贴图片后自动触发附件上传流程（与手动上传同链路）
3. 粘贴成功后在编辑区域显示图片结果（并插入附件引用）
4. 失败时提供明确提示，不影响用户继续输入文本
5. 保持多文件模式与单文件模式下行为一致

## 3. 现状与问题

### 3.1 当前粘贴行为

- 输入区的粘贴主要面向文本
- 图片粘贴时不会被识别为附件上传动作
- 用户无法通过截图后直接粘贴完成记录

### 3.2 当前附件能力

- 已有附件上传能力（文件选择后上传）
- 已有附件命名与保存策略（附件目录、重名处理、链接生成）
- 但该能力尚未被“粘贴事件”复用

## 4. 范围

### 4.1 In Scope

- 快速记录输入区域的粘贴事件监听
- 从剪贴板识别图片文件（`image/*`）
- 调用现有附件上传函数保存图片到附件目录
- 将上传成功的图片以当前插件约定格式插入到记录内容中，并在编辑区域可见
- 多图粘贴（同一次粘贴内包含多张图片）按顺序处理
- 上传失败通知与不中断文本编辑

### 4.2 Out of Scope

- 图片压缩/裁剪/格式转换（本期不新增）
- 拖拽上传体验改造
- 富文本编辑器中的图片预览能力
- 云端图床上传（本期仅 Vault 附件流程）

## 5. 功能需求

## 5.1 FR-1 粘贴事件捕获

- 快速记录输入区必须监听 `paste` 事件
- 当剪贴板包含图片文件时，拦截默认粘贴行为
- 当仅有文本时，保持原有文本粘贴行为不变

## 5.2 FR-2 粘贴图片走附件上传流程

- 每张粘贴图片必须调用已有附件上传链路
- 保存路径、文件命名、重名处理与手动上传保持一致
- 上传过程中的异常统一走现有错误处理规范

## 5.3 FR-3 内容插入与顺序

- 图片上传成功后，向当前记录正文插入附件引用
- 引用格式与现有附件插入格式保持一致（不新增新语法）
- 多图时按剪贴板文件顺序依次插入
- 插入完成后，编辑区域必须立即可见该图片结果（至少可见图片 Markdown 引用；若存在预览区则同步显示预览）

## 5.4 FR-4 失败处理

- 某张图片上传失败时：
  - 给出用户可见错误提示（Notice 或等效提示）
  - 不阻塞后续图片处理（如果有）
  - 不影响当前已输入文本
- 全部上传失败时，文本输入仍可继续并正常保存

## 5.5 FR-5 一致性与兼容性

- 多文件模式与单文件模式行为一致
- 不改变现有保存记录主流程
- 不破坏已有手动附件上传入口

## 6. 交互与用户体验要求

1. 用户在快速记录输入区粘贴截图后，能够立即在编辑区域看到图片结果（引用或预览）
2. 成功反馈应轻量，不干扰持续输入
3. 失败反馈应明确指出“图片上传失败”，便于重试
4. 粘贴图片不应导致输入框失焦或内容被整体覆盖

## 7. 非功能需求

## 7.1 性能

- 单张常见截图粘贴应在可接受时间内完成（主观无明显卡顿）
- 多图粘贴场景下应避免阻塞 UI（可串行异步处理）

## 7.2 稳定性

- 任一图片处理异常不应导致快速记录组件崩溃
- 出错后插件应保持可继续输入与可保存状态

## 7.3 可维护性

- 优先复用既有附件处理函数，避免新建重复上传逻辑
- 粘贴入口仅负责“识别 + 调用”，存储规则仍由附件模块统一管理

## 8. 验收标准（MVP）

1. 在快速记录输入区复制图片后粘贴，能够自动保存到附件目录
2. 粘贴成功后自动插入与手动上传一致的图片引用，并在编辑区域立即可见
3. 连续粘贴多张图片时，均可成功上传并按顺序插入
4. 粘贴文本时行为与当前版本一致，不受影响
5. 上传失败时有错误提示，且用户仍可继续输入并保存记录
6. 在单文件/多文件模式下均通过上述用例

## 9. 风险与注意事项

- 不同平台（macOS / Windows）剪贴板图片 MIME 类型可能存在差异，需按 `image/*` 宽匹配
- 若只读取 `files[0]` 可能漏掉多图粘贴，需遍历剪贴板文件集合
- 粘贴事件拦截过宽会影响文本粘贴，必须严格区分“有图片”与“仅文本”
- 附件目录不存在、权限不足或文件创建失败时需保证失败可恢复

## 10. 实施建议（开发顺序）

1. 在快速记录输入控件接入 `paste` 事件监听
2. 抽取剪贴板图片列表识别逻辑（支持多图）
3. 逐张调用现有附件上传函数并收集返回路径
4. 统一插入附件引用到正文，补齐成功/失败提示
5. 回归验证：文本粘贴、单图、多图、失败场景、两种存储模式
